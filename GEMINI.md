# Project: Curso Engenheiro de Software IA (Produtividade 10%)

This repository serves as the central hub for the "Engenheiro de Software na Era da IA" course materials. It is a documentation-centric project designed to guide developers through 7 modules focused on leveraging the **Gemini CLI** to achieve 10% productivity in software engineering.

## 📁 Directory Overview

The project is structured as a curriculum, with content organized chronologically and by topic:

- **`encontros/`**: The core of the project. Contains 7 directories, each representing a specific lesson.
  - Each directory typically includes a `README.md` and a `cronograma.md`.
  - Content typically includes:
    - **Tecnologias Recomendadas**: Focus on **Gemini CLI** and sub-agents.
    - **Teoria**: Conceptual background on the shift from "code writer" to "architect/reviewer" using CLI-first workflows.
    - **Exemplos Práticos**: Specific prompts and use cases for the terminal.
    - **Exercício Prático**: Hands-on tasks for students.
- **`CURSO_ENGENHEIRO_IA.md`**: The master syllabus. It outlines the 7-meeting roadmap, module objectives, and the overall teaching methodology.
- **`README.md`**: High-level introduction and repository navigation guide.

## 🎯 Key Concepts & Conventions

- **CLI-First & Agentic Workflow**: The project emphasizes using **Gemini CLI** not just for code completion, but for full-lifecycle agentic development (architecture, TDD, debugging, refactoring, and DevOps).
- **Context Management**: A recurring theme is that "Context is King," focusing on how the Gemini CLI indexes and understands entire repositories via terminal.
- **Agentic Development**: Focus on sub-agents and autonomous execution within the CLI environment.
- **Language**: All course materials are written in **Portuguese (PT-BR)**.

## 🚀 Usage for Gemini CLI

When interacting with this repository, keep the following in mind:

1.  **Reference the Syllabus**: Use `CURSO_ENGENHEIRO_IA.md` to understand where a specific topic fits within the 7-meeting program.
2.  **Module Deep Dives**: For detailed information on specific workflows, look into the corresponding directory in the `encontros/` folder.
3.  **Non-Code Nature**: This is primarily a documentation repo. While it discusses code, its primary purpose is instructional delivery and curriculum management.
4.  **Instructional Context**: Treat the contents of `encontros/` as the "source of truth" for the methodologies being taught (e.g., specific prompt engineering techniques or architectural patterns recommended by the course).
