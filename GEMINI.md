# Project: Curso Engenheiro de Software IA (Produtividade 10x)

This repository serves as the central hub for the "Engenheiro de Software na Era da IA" course materials. It is a documentation-centric project designed to guide developers through 12 modules focused on leveraging AI to achieve 10x productivity in software engineering.

## 📁 Directory Overview

The project is structured as a curriculum, with content organized chronologically and by topic:

- **`encontros/`**: The core of the project. Contains 12 Markdown files (`01-*.md` to `12-*.md`), each representing a specific lesson.
  - Each file typically includes:
    - **Tecnologias Recomendadas**: Tools like Cursor, GitHub Copilot, Claude, and specialized AI agents.
    - **Teoria**: Conceptual background on the shift from "code writer" to "architect/reviewer."
    - **Exemplos Práticos**: Specific prompts and use cases.
    - **Sugestões de Ferramentas**: Extensions, CLI tools (like Aider or GPT-Engineer), and configurations.
    - **Exercício Prático**: Hands-on tasks for students.
- **`CURSO_ENGENHEIRO_IA.md`**: The master syllabus. It outlines the 12-week roadmap, module objectives, and the overall teaching methodology.
- **`README.md`**: High-level introduction and repository navigation guide.

## 🎯 Key Concepts & Conventions

- **AI-First Workflow**: The project emphasizes using AI not just for code completion, but for architecture, TDD, debugging, refactoring, and DevOps.
- **Context Management**: A recurring theme is that "Context is King," focusing on tools like Cursor that index entire repositories.
- **Agentic Development**: The latter part of the course introduces autonomous agents (Aider, OpenDevin) as the future of software work.
- **Language**: All course materials are written in **Portuguese (PT-BR)**.

## 🚀 Usage for Gemini CLI

When interacting with this repository, keep the following in mind:

1.  **Reference the Syllabus**: Use `CURSO_ENGENHEIRO_IA.md` to understand where a specific topic fits within the 12-week program.
2.  **Module Deep Dives**: For detailed information on specific tools (e.g., Cursor, Playwright, Mermaid.js), look into the corresponding file in the `encontros/` directory.
3.  **Non-Code Nature**: This is primarily a documentation repo. While it discusses code, its primary purpose is instructional delivery and curriculum management.
4.  **Instructional Context**: Treat the contents of `encontros/` as the "source of truth" for the methodologies being taught (e.g., specific prompt engineering techniques or architectural patterns recommended by the course).
